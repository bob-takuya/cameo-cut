# Silhouette Cameo 5 座標系の理解

## 元のSilhouette Studioの動作

### 1. **絶対座標系を使用**
- デバイスの物理的な原点（左上）を基準とした絶対座標
- ホームポジション(0,0)は常にデバイスの左上隅
- リアルタイムの位置追跡は**行わない**

### 2. **ユーザーワークフロー**
```
1. マテリアルをカッティングマットに配置
2. マットをデバイスにセット
3. デバイスのボタンで必要に応じて原点調整
4. ソフトウェアでデザインを配置
5. 送信 → デバイスはホームから開始して絶対座標で移動
```

### 3. **レジストレーションマーク機能**
- プリント済みのマークを検出
- デバイスが自動的にマークを読み取り、オフセットを補正
- これにより手動位置調整が不要に

## 現在の実装の問題点

### 問題
1. `design_offset`（キャンバス上のドラッグ）を座標計算に使用
2. デバイスを手動で動かすとソフトウェアが追跡できない
3. 複雑なオフセット計算: `final_offset = design_offset - dxf_min`

### 結果
- プレビューと実際のカット位置がずれる
- デバイスを手動で動かすと完全に破綻

## 推奨される修正

### シンプルな座標系
```python
# DXFを(0,0)から始まるように正規化するだけ
bounds = entities.get_bounding_box()
offset_x = -bounds.min_x  # DXFの最小値を引く
offset_y = -bounds.min_y  # これで(0,0)から始まる

# design_offsetは使わない！
```

### ユーザーワークフロー
```
1. DXFファイルを読み込む
2. プレビューで確認（常に(0,0)から）
3. デバイスのボタンで物理的な開始位置を決定
4. 送信ボタン → ジョブはH（ホーム）コマンドから開始
```

### 利点
- シンプル
- デバイスの手動移動に影響されない
- 予測可能な動作
- Silhouette Studioと同じ方式

## 位置クエリについて

OAコマンド（Output Actual position）は標準HPGLコマンドですが、
Cameo 5がサポートしているかは不明です。

テストスクリプト `test_position_query.py` で確認できます:
```bash
python3 test_position_query.py
```

サポートされていない場合:
- リアルタイム位置追跡は諦める
- 代わりに「テストカット」機能を提供
- ユーザーが手動で位置を確認
